### Bloco 1: Definição e Modelagem Estrutural (Foco em "Modelagem das Classes: 1,5 pts")

Este bloco garante a base teórica e estrutural necessária para o relatório e para o início da codificação em Java.

* **Sub-bloco 1.1: Escopo e Requisitos Iniciais**
* **Tarefa 1.1.1: Definição do Produto**
* *Sub-tarefa:* Opção 2 (Aplicação Livre): Sistema Bancário.
* *Sub-tarefa:* Java (JDK) versão 26.0.1 e a IDE VS Cod.

* **Tarefa 1.1.2: Planejamento de Pontuação Extra (+0,5)**
* *Sub-tarefa:* Decidir a interface: gráfica usando `Java Swing`.
* *Sub-tarefa:* Planejar a persistência de dados: salvar as transações e contas em um arquivo `.csv`.

* **Sub-bloco 1.2: Engenharia do Diagrama UML**
* **Tarefa 1.2.1: Mapeamento de Classes Básicas (Mínimo de 3)**
* *Sub-tarefa:* Listar as classes principais. Sugestão: `Cliente`, `Conta` e `Banco`.
* *Sub-tarefa:* Definir os atributos (ex: `nome`, `cpf`, `saldo`, `numeroConta`) e métodos (ex: `depositar()`, `sacar()`, `transferir()`).

* **Tarefa 1.2.2: Incorporação de Herança, Polimorfismo e Interfaces (Bônus/Extra)**
* *Sub-tarefa:* Projetar Herança: Criar classes filhas `ContaCorrente` (com limite/cheque especial) e `ContaPoupanca` (com rendimento) que herdam de `Conta`.
* *Sub-tarefa:* Projetar Polimorfismo: Sobrescrever (override) o método `sacar()` para que a `ContaCorrente` cobre uma taxa ou acesse o limite.
* *Sub-tarefa:* Projetar Interfaces: Criar uma interface `Autenticavel` para permitir que clientes e funcionários façam login.

* **Tarefa 1.2.3: Desenho Final do Diagrama**
* *Sub-tarefa:* Criar o diagrama simples UML detalhando as associações (um `Cliente` tem uma `Conta`, o `Banco` gerencia várias `Contas`).

### Bloco 2: Construção da Lógica e Aplicação POO

O núcleo do desenvolvimento, onde as abstrações do UML se transformam em código Java funcional.

* **Sub-bloco 2.1: Esqueleto Orientado a Objetos em Java**
* **Tarefa 2.1.1: Classes e Construtores**
* *Sub-tarefa:* Criar os arquivos `.java` para cada classe desenhada.
* *Sub-tarefa:* Implementar os construtores para inicializar os objetos corretamente (ex: toda `Conta` nova começa instanciando um `Cliente` e com saldo zero).

* **Tarefa 2.1.2: Encapsulamento Restrito**
* *Sub-tarefa:* Declarar todos os atributos essenciais como `private` (ou `protected` para herança).
* *Sub-tarefa:* Gerar os métodos `getters` e `setters` necessários, proibindo alterações diretas de variáveis críticas (ex: não deve haver `setSaldo()`, o saldo só muda via depósito ou saque).




* **Sub-bloco 2.2: Implementação da Lógica Bancária**
* **Tarefa 2.2.1: Desenvolvimento das Regras de Negócio**
* *Sub-tarefa:* Implementar estruturas de decisão (`if/else`) para validar operações (ex: impedir saque se o valor for maior que o saldo + limite).
* *Sub-tarefa:* Implementar o loop principal de execução (ex: um `while` ou `do-while` que mantém o menu do banco ativo até o usuário escolher "Sair").


* **Tarefa 2.2.2: Codificação dos Recursos Avançados**
* *Sub-tarefa:* Utilizar a palavra-chave `extends` para configurar as classes filhas e `@Override` para o polimorfismo.
* *Sub-tarefa:* Utilizar a palavra-chave `implements` para a interface `Autenticavel`.
